# v0.1.5 Preparation Plan

This plan details proactive enhancements to the UX, stream visuals, and system robustness before user feedback arrives.

## Objective
To elevate the broadcast quality with rotating leaderboards, protect session data with periodic saves, and prevent visual flooding from low-value spam events.

## Scope & Impact
- **Visuals:** The overlay leaderboard will automatically rotate between Top Cheerers and Top Gifters every 30 seconds.
- **Robustness:** `state.json` will flush to disk every 5 minutes to prevent data loss on sudden power loss or crashes.
- **Throttling:** 1-bit cheers will be throttled slightly to prevent chatters from flooding the screen with micro-cheers.
- **Asset UX:** The dashboard configuration editors will use a dropdown of available sounds from the `assets/sounds/` directory instead of requiring manual text entry.

## Implementation Steps

### Phase 1: Periodic State Persistence
1. **`server/state.js`:**
   - Add a `setInterval` that calls `this.flush()` every 5 minutes (300,000ms).

### Phase 2: Throttling 1-Bit Cheers
1. **`server/pipeline/throttler.js`:**
   - Enhance the throttler to inspect the payload.
   - If `event.type === 'cheer'` and `event.payload.bits < 10`, apply a 2000ms throttle.

### Phase 3: Rotating Overlay Leaderboard
1. **`overlay.html`:**
   - Modify `renderLeaderboard(lb)` to store the leaderboard data in a local variable.
   - Implement a `setInterval` that toggles the displayed list between `lb.bits` and `lb.gifts` every 15 seconds.
   - Update the `.lb-title` to reflect the current category (e.g., "Top Cheerers" vs "Top Gifters").

### Phase 4: Asset Discovery API
1. **`server/index.js`:**
   - Create a `GET /api/assets` endpoint that returns a list of files in `assets/sounds/`.
2. **`dashboard/app.js` & `dashboard/index.html`:**
   - Fetch `/api/assets` on load.
   - Modify `addGoalConfig`, `addRedeemConfig`, and `renderConfigEditors` to use a `<select>` dropdown populated with the available sounds, rather than a text `<input>`.

## Verification
- Start the server and verify `/api/assets` returns the correct sound files (if any exist).
- Add fake data to `lb.gifts` and `lb.bits` via the dashboard console and verify the overlay rotates them.
- Spam `testEvent('cheer', {bits: 1})` and verify they are throttled compared to `testEvent('cheer', {bits: 100})`.
